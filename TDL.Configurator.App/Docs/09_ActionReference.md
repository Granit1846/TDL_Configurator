# Action Reference: что делает каждая команда (кратко и по делу)

Этот файл — «человеческие» описания ключевых ACTION.
Каталог (просто список) — в 03_ActionCatalog_NORMAL.md.

Формат NORMAL:
```txt
tdl_send.exe NORMAL <ACTION> [SOURCE]
```

---

## SYSTEM_PING
**Что делает:** проверяет, что `tdl_send.exe` видит pipe и плагин принимает команды.  
**Когда полезно:** всегда, это первое действие для диагностики.  
**Если не работает:** см. 11_Logs_Diagnostics.md (обычно pipe не найден или плагин не загрузился).  
**Пример:**
```txt
tdl_send.exe NORMAL SYSTEM_PING 2
```

## SYSTEM_HEALING
**Что делает:** лечит/восстанавливает игрока (видимый эффект).  
**Когда может не сработать:** если действие отключено/заблокировано вашей сборкой.  
**Пример:**
```txt
tdl_send.exe NORMAL SYSTEM_HEALING 2
```

## SYSTEM_BLESSING
**Что делает:** даёт благословение/бафф.  
**Комментарий:** эффект зависит от реализации мода (какой именно бафф).  

---

## INVENTORY_SCATTER
**Что делает:** «рассыпает» часть/весь инвентарь вокруг игрока (drop‑серия).  
**Риски:** высокая нагрузка, если большие значения количества/радиуса/малый интервал.  
**Связанные настройки INI:** `[Inventory]` (см. 10_INI_Settings_Reference.md).  
**Пример:**
```txt
tdl_send.exe NORMAL INVENTORY_SCATTER 2
```

## INVENTORY_DROP_ALL
**Что делает:** сбрасывает весь инвентарь.  
**Риски:** крайне разрушительно для прохождения; опасно в BURST.  
**Рекомендуется:** использовать только сознательно, лучше один раз и с защитой токенов.  

---

## TELEPORT_* (телепорты)
**Что делает:** телепортирует игрока в заранее заданную точку или выбирает случайную.  
**Риски:** ломает сцены/квесты, если телепортировать в неподходящий момент.  
**Рекомендуется:** только после загрузки сохранения, не в диалоге, не в катсценах.

**Примеры:**
```txt
tdl_send.exe NORMAL TELEPORT_RANDOM_CITY 2
tdl_send.exe NORMAL TELEPORT_RANDOM_DANGER 2
tdl_send.exe NORMAL TELEPORT_WHITERUN 2
```

---

## SUMMON_* (призывы)
**Что делает:** призывает существо/НПС.  
**Риски:** нагрузка на AI, особенно если много и быстро.  
**Рекомендуется:** серии — только аккуратно (см. 04_FORCE_Safety.md).

**Примеры:**
```txt
tdl_send.exe NORMAL SUMMON_ANY 2
tdl_send.exe NORMAL SUMMON_DRAGON 2
```

---

## VIRUS_* (болезни/проклятия)
**VIRUS_DISEASE:** накладывает болезнь/дебафф.  
**VIRUS_VAMPIRE:** заражение вампиризмом (может влиять на прохождение).  
**VIRUS_WEREWOLF:** заражение оборотнем (аналогично).

**Риски:** эти действия изменяют «статус» персонажа. Делайте это осознанно.  

---

## WEATHER_* (погода)
**Что делает:** меняет погоду (на улице эффект заметен сильнее).  
**Особенности:** в интерьерах может быть минимальный эффект.  

**Пример:**
```txt
tdl_send.exe NORMAL WEATHER_STORM 2
```

---

## CHAOS_BACKFIRE / CHAOS_LOW_G
**Что делает:** включает хаотичные эффекты (backfire, физика и т.п.).  
**Риски:** сильные значения силы/радиуса могут приводить к «полёту через карту» и багам физики.  
**Связанные настройки INI:** `[Chaos]` (см. 10).

---

## WRATH_11 / WRATH_12 / WRATH_13
**Что делает:** запускает один из пресетов Wrath.  
**Комментарий:** различия между 11/12/13 задаются вашей сборкой.  
**Связанные настройки INI:** `[Wrath]` (см. 10).

---

## GIGANT_* (размер/скорость)
- `GIGANT_BIG` / `GIGANT_SMALL` — изменение размера
- `GIGANT_SPEED` / `GIGANT_SLOW` — скорость
- `GIGANT_RESET` — сброс

**Связанные настройки INI:** `[Gigant]` (см. 10).  
**Риски:** экстремальные значения могут ломать коллизии/анимации.

---

## HUNTER_START
**Что делает:** запускает сценарий Hunter.  
**Связанные настройки INI:** `[Hunter]` (см. 10).  

---

## COMEDY_* (сценарии)
`COMEDY_ARENA`, `COMEDY_ESCORT`, `COMEDY_FAKE_HERO`, `COMEDY_HORROR`  
**Что делает:** запускает сценарий/ивент.  
**Комментарий:** детали зависят от сборки; смотрите подсказки UI и параметры `[Comedy]`.
